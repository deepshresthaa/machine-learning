## `max_depth`
### Without hyperparameter tuning, the DecisionTreeClassifier chose criterion as "gini" and max_depth=None. Here, max_depth=None means capture all if-else condition.
### but doing this might lead to overfitting, if the model start to capture all small rules that might be mis-placed or outliers, then decision tree will capture that too, so when testing on some data points near the misplaced region, it will predict the wrong class. 
### and , when max_depth is set to lower value then it will not consider enough if-else condition and tend to underfit. 
### so choosing approprite value of `max_depth` is crucial.

## `criterion`
### sometimes `criterion="entropy"` performs better than `criterion="gini"`, by default it is set `gini`, so to change this do `DecisionTreeClassifier(criterion="entropy", max_depth="3")`, 

## On hyperparameter tuning on `criterion` and `max_depth` for `iris_dataset`, model improved the accuracy from `86.25 % accuracy score` to `91.25% accuracy score`.
